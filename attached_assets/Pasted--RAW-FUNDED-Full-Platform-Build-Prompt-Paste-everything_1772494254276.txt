## SUPABASE SCHEMA

```sql
-- Profiles
create table profiles (
  id uuid references auth.users primary key,
  email text,
  username text unique,
  full_name text,
  tier text default 'unverified',
  -- unverified | verified | elite | titan | banned
  tries_used int default 0,
  -- unverified only: increments when account hits zero
  tries_visible bool default false,
  -- flips to true after first account zero
  banned_at timestamp,
  ban_expires_at timestamp,
  -- auto-set to banned_at + 6 months
  balance numeric default 10000,
  -- starting demo balance
  created_at timestamp default now()
);

-- Trades
create table trades (
  id uuid primary key default gen_random_uuid(),
  account_id uuid references profiles(id),
  instrument text,
  -- XAUUSD_m, XAGUSD_m, MNQ_m, MES_m (micro)
  -- XAUUSD, XAGUSD, MNQ, MES (standard - verified+)
  side text, -- BUY | SELL
  size numeric, -- lot size
  open_price numeric,
  close_price numeric,
  pnl numeric default 0,
  status text default 'pending',
  -- pending | open | closed | error
  mt5_ticket bigint,
  opened_at timestamp default now(),
  closed_at timestamp
);

-- Verifications
create table verifications (
  id uuid primary key default gen_random_uuid(),
  user_id uuid references profiles(id),
  prop_firm text,
  -- FTMO | TopStep | MyForexFunds | Funded Next | TFT | Apex | Earn2Trade | Leeloo | Lark | Lucid | Other
  proof_type text, -- screenshot | url | email_forward | certificate
  proof_url text,
  -- Supabase storage URL after upload
  payout_count int default 0,
  -- 0 = just passed, 1 = Elite, 2-3 = Titan
  notes text,
  status text default 'pending',
  -- pending | approved | rejected
  submitted_at timestamp default now(),
  reviewed_at timestamp,
  reviewed_by text
);

-- Withdrawals
create table withdrawals (
  id uuid primary key default gen_random_uuid(),
  user_id uuid references profiles(id),
  amount numeric,
  method text, -- wise | usdc | usdt
  destination text,
  -- email for Wise, wallet address for crypto
  status text default 'pending',
  -- pending | processing | completed | rejected
  requested_at timestamp default now(),
  processed_at timestamp
);

-- Ban appeals
create table ban_appeals (
  id uuid primary key default gen_random_uuid(),
  user_id uuid references profiles(id),
  proof_type text,
  -- profitable_track_record | passed_eval
  proof_url text,
  notes text,
  status text default 'pending',
  -- pending | approved | rejected
  submitted_at timestamp default now()
);
```

## TIER SYSTEM — ENFORCE IN ALL TRADE LOGIC

```javascript
const TIERS = {
  unverified: {
    leverage: 50,
    maxLot: 0.01,
    instruments: ['XAUUSD_m', 'XAGUSD_m', 'MNQ_m', 'MES_m'],
    // Micro only
    tries: 3,
    // account zero = try used
    label: 'Unverified',
    color: '#71717A'
  },
  verified: {
    leverage: 250,
    maxLot: 0.10,
    instruments: ['XAUUSD_m', 'XAGUSD_m', 'MNQ_m', 'MES_m',
                  'XAUUSD', 'XAGUSD', 'MNQ', 'MES'],
    tries: null,
    // unlimited
    label: 'Verified',
    color: '#3B82F6'
  },
  elite: {
    leverage: 500,
    maxLot: 0.50,
    instruments: ['XAUUSD_m', 'XAGUSD_m', 'MNQ_m', 'MES_m',
                  'XAUUSD', 'XAGUSD', 'MNQ', 'MES'],
    tries: null,
    label: 'Elite',
    color: '#E8C547'
  },
  titan: {
    leverage: 2000,
    maxLot: 999,
    // no limit
    instruments: ['XAUUSD_m', 'XAGUSD_m', 'MNQ_m', 'MES_m',
                  'XAUUSD', 'XAGUSD', 'MNQ', 'MES', 'EURUSD', 'GBPUSD'],
    tries: null,
    label: 'Titan',
    color: '#22C55E'
  },
  banned: {
    leverage: 0,
    maxLot: 0,
    instruments: [],
    label: 'Banned',
    color: '#EF4444'
  }
}
```

**Ban logic (enforce server-side in API route):**
- When a trade closes and account balance hits 0 or below → increment `tries_used`, set `tries_visible = true`
- If `tries_used >= 3` → set `tier = 'banned'`, set `banned_at = now()`, set `ban_expires_at = now() + 6 months`
- After first try used → show try counter in dashboard: "Try 2 of 3" with yellow warning
- Banned users see ban screen with expiry date and appeal option

**Trade validation (before any order):**
```javascript
// Check tier limits before accepting trade
if (trade.size > TIERS[user.tier].maxLot) {
  return error('Lot size exceeds your tier limit. Verify your funded status to unlock larger positions.')
}
if (!TIERS[user.tier].instruments.includes(trade.instrument)) {
  return error('This instrument requires a higher tier. Verify your funded status.')
}
if (user.tier === 'banned') {
  return error('Account suspended. Check your verification status.')
}
```

## API ROUTES TO BUILD

```
POST /api/trades/open      — validate tier limits, insert trade
POST /api/trades/close     — close trade, calculate P&L, check if balance hit zero
POST /api/verify/submit    — upload proof to storage, insert verification row
POST /api/withdraw/request — validate amount, insert withdrawal row
POST /api/appeal/submit    — insert ban appeal row
GET  /api/leaderboard      — fetch verified+ traders with aggregated stats
```

**In `/api/trades/close` — ban logic:**
```javascript
// After closing trade and updating balance
if (newBalance <= 0 && user.tier === 'unverified') {
  await supabase.from('profiles').update({
    tries_used: user.tries_used + 1,
    tries_visible: true,
    balance: 10000 // reset balance for next try
  }).eq('id', user.id)
  
  if (user.tries_used + 1 >= 3) {
    const banExpiry = new Date()
    banExpiry.setMonth(banExpiry.getMonth() + 6)
    await supabase.from('profiles').update({
      tier: 'banned',
      banned_at: new Date().toISOString(),
      ban_expires_at: banExpiry.toISOString()
    }).eq('id', user.id)
  }
}
```
